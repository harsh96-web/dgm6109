About This Project

I tracked my daily fitness and wellness habits over 12 consecutive days from September 25 to October 6, 2025. The goal was to explore potential relationships between physical activity, nutrition, sleep quality, and overall restfulness.

Data Collected:

  • Workout Duration: Minutes spent exercising (0-102 min)
  • Sleep Hours: Total sleep per night (6.0-8.5 hrs)
  • Protein Intake: Daily protein consumption (110-150g)
  • Restfulness: Sleep quality rating 1-5 scale

Below are three different visualization proposals exploring these metrics from different analytical perspectives.

Sketch 1: Protein Intake vs Sleep Quality

Hypothesis

As protein intake (grams) increases, sleep hours (hours) also increase, suggesting that better nutrition supports longer sleep duration. Higher workout duration (minutes) is associated with better sleep quality, shown through bubble sizeâ€”longer, more intense workouts appear to promote deeper rest. Restfulness levels (1-5 scale) are indicated by color from red (very tired) to dark green (excellent rest), allowing quick visual identification of peak recovery days.

The color gradient from red through yellow to dark green represents restfulness levels, making it easy to identify which days I felt most rested.

Sketch 3: Battery Energy Dashboard

Creative Research Question

How do all four fitness metrics improve over 12 days, and what does optimal daily fitness "energy" look like? This creative visualization uses the battery metaphor to show daily "energy levels" across multiple dimensions.

Data Elements (All Four Metrics!)

  • Battery Width: Protein Intake (grams) â€¢ Thin (110g) to Wide (150g)
  • Fill Level: Sleep Hours (%) â€¢ 60% (6.0 hrs) to 85% (8.5 hrs)
  • Blue Dots: Workout Duration (minutes) â€¢ 0 dots (rest day) to 5 dots (102 min)
  • Fill Color: Restfulness (1-5) â€¢ Red (tired) to Dark Green (excellent)

Visual Representation

This creative visualization uses battery icons as a metaphor for "charging up" fitness levels, inspired by the Dear Data project's approach to representing personal data through familiar, everyday objects. Each of the 12 batteries represents one day, with four visual dimensions encoding different metrics simultaneously. Width increases with protein intake, fill level represents sleep as a percentage of maximum, blue dots above show workout duration (more dots = longer workouts, no dots = rest day), and the fill color indicates restfulness level. The progression from Day 1 to Day 12 clearly shows improvement across all four metrics, with the final "peak" day demonstrating optimal fitness energy: wide battery (high protein), nearly full (excellent sleep), multiple dots (intense workout), and dark green color (maximum restfulness).

Design Resources and How I Used Them

Dear Data by Giorgia Lupi and Stefanie Posavec (http://www.dear-data.com/theproject)

This project helped me think about creative ways to visualize personal data in Sketch 3. Specifically, I was inspired by how they use everyday objects and metaphors to represent dataâ€”like using weather symbols for emotions. This gave me the idea to use battery icons as a metaphor for "charging up" my fitness levels over 12 days, with each battery showing all four metrics simultaneously (width for protein, fill level for sleep, dots for workout duration, and color for restfulness). Without this resource, I probably would have created a standard chart instead of the creative battery visualization.

Data Visualisation Catalogue (https://datavizcatalogue.com/index.html)

I used the "Search By Function" page to find visualization types that work for comparing multiple variables over time. This helped me decide on a multi-line chart for Sketch 2 because the catalogue explained that line charts are good for showing trends and patterns when you have multiple data series sharing the same time-based axis. This confirmed that putting all three metrics on one chart would make it easier to see if they move together or separately over the 12-day period.

Observable D3 Gallery (https://observablehq.com/@d3/gallery)

I looked at bubble chart examples to understand how to encode multiple dimensions in a scatterplot for Sketch 1. Specifically, I learned that using different bubble sizes requires careful scalingâ€”the examples showed me that bubble area (not diameter) should be proportional to the data value. I also learned from these examples that using some transparency on bubbles helps when they overlap, which is why I made note of that in my sketch design.

Class lecture on proportional ink and data visualization principles

The lecture's discussion of the "rule of proportional ink" directly influenced how I designed bubble sizes in Sketch 1 and fill percentages in Sketch 3. This rule says that the amount of ink (or color) should be proportional to the data value it represents. For the battery visualization, this meant using proportional fill percentage (like 71% for 6.0 hours representing 71% of the maximum 8.5 hours) rather than arbitrary heights, ensuring an accurate visual representation.

From Data to Viz (https://www.data-to-viz.com/)

I used the decision tree tool on this website to verify my visualization type choices. Starting from "two numerical variables" led me to confirm that scatterplots work well for Sketch 1's protein-sleep relationship. For Sketch 2, I followed the path for "one categorical (time) and several numerical variables" which confirmed that multi-line charts are appropriate for showing multiple metrics over the same time period.
